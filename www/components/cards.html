Cards

Cards are used to display information in a visual, scannable manner. Use in scenarios when users want to associate an image with data, such as a Project or Site, or in association with user accounts where it is helpful to associate an image with a person using the system.

Common card #

Include the base class .tn-card on all cards.

The common card is static and non-interactive. It is an adaptable container well-suited to present collection items.

Button card #

Add the ARIA attribute role="button" or class .tn-card--button.

The button card includes hover effects and uses link colors to indicate interactivity.

Accessibility: The ARIA role of "button" is appropriate when a the entire card is clickable.

Disabled card #

Add the ARIA attribute aria-disabled="true" or class .is-disabled.

Toggle the ARIA attribute or the class name to switch the interactive state of any card.

Accessibility: Use 'aria-disabled' when the control is interactive but not currently available, e.g., an Edit Mode is enabled in a view which enables the control. The class 'is-disabled' is approriate when state is presentational and interactivity is not expected, e.g., an item is discontinued and will not have a click event.

Actions card #

Include additional element .tn-card__actions.

If a card has more than one action, an additional container can be added after the content container.

Accessibility: The AIRA role of "button" should not be used in this case since the entire card is not interactive.
