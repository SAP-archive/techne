Button card #

Add the ARIA attribute role="button" or class .tn-{{ key }}--button.

The button card includes hover effects and uses link colors to indicate interactivity.

Accessibility: The ARIA role of "button" is appropriate when a the entire card is clickable.

{% set example %} {{ card( modifier="button", button=true, title="Lorem ipsum dolor quam placeat numquam repellendus", description="Tenetur quam placeat obcaecati molestias voluptatibus dolorum ut.", status='Voluptate quis adipisci sit' ) }} {% endset %}

{{ example }}

Disabled card #

Add the ARIA attribute aria-disabled="true" or class .is-disabled.

Toggle the ARIA attribute or the class name to switch the interactive state of any card.

Accessibility: Use 'aria-disabled' when the control is interactive but not currently available, e.g., an Edit Mode is enabled in a view which enables the control. The class 'is-disabled' is approriate when state is presentational and interactivity is not expected, e.g., an item is discontinued and will not have a click event.

{% set example %} {{ card( button=true, title='Repellendus obcaecati molestias', description='Lorem ipsum dolor sit amet, consectetur adipisicing elit aute irure dolor in reprehenderit in voluptate', status='Voluptate quis adipisci sit | Voluptate quis adipisci sit', disabled=true ) }} {% endset %}

{{ example }}

Actions card #

Include additional element .tn-{{ key }}__actions.

If a card has more than one action, an additional container can be added after the content container.

Accessibility: The AIRA role of "button" should not be used in this case since the entire card is not interactive.

{% set example %} {{ card( modifier="button", title="Lorem ipsum dolor quam placeat numquam repellendus", description="Tenetur quam placeat ipsum dolor sit amet obcaecati molestias voluptatibus dolorum ut.", status='Voluptate quis adipisci sit', actions='
•••
' ) }} {% endset %}
