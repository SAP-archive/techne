8 Localization Editor

The Localization component is used when any content needs to allow for translation into a multiple languages.

8.1 Localization Editor

Case description:
The localization tooling will present tabs for each localization language supported. The first and fall back language will be the first tab and will auto-selected. The selected tab is presented with a blue background. The non-selected tabs are presented with white background. The start informs if language is mandatory.

8.4 Localization Editor

More menu:
The number of recommended is 5. If more tabs are required, displayed a more menu replacing the 5th item. The more Menu dropdown lists more available localization languages. The label "More" reflects mandatory fields from the list. In this case there are hidden tabs with mandatory fields, so the "More" label is followed by a start.
Error handling Scenario:
The outcome of a validation error for a language hidden on the More menu is present “More” content in red and append the exclamation mark.

8.6 Localization Editor

More menu:
When user selects an item from the More Menu the hidden tab gets highlighted in blue and replaces the MORE label. The arrow is kept next to it to remind user that the selected options is part of the More menu.
Error handling Scenario 1:
If error happens to the last selected tab from more menu, that tab remains open and its background turns to red. If user opens dropdown he will see that one option on the menu presented on red.
Error handling Scenario 2: (Recommendation)
If an error happens to a hidden tab on more menu, but the last selected tab was also an option of the more menu, the system resets the selected tab and switch to the one that contains the error.
Error handling Scenario 3: (Recommendation)
If errors happen to more than one hidden tab under the more menu, but the last selected tab was also an option of the more menu, the system resets the selected tab to the default tab (EN in this case) and shows the More menu in red appended by an exclamation mark.
